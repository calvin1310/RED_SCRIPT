01   Histoire
L'ARCHIVE
001 / LE MONDE
L'ARCHIVE

Un lieu construit pour écrire la réalité. Les Architectes ont découvert que le monde n'est pas fait de matière — il est fait de définitions. Ils ont appris à les modifier. Puis tout s'est arrêté.

002 / LE PERSONNAGE
LA SILHOUETTE

Tu n'as pas de nom. Tu es apparu parce que quelqu'un a commencé une phrase et ne l'a jamais terminée. Tu es instable. Tu glitches. Tu deviens rouge. Tu es un brouillon.

003 / LE BUT
LE TERMINAL

Au fond de l'Archive existe le Script Terminal. Il attend une phrase. C'est toi qui l'écriras. Ce que tu écris détermine la fin du monde.

004 / LES FRAGMENTS
PHRASES PERDUES

18 phrases inachevées disséminées dans l'Archive. Des tentatives abandonnées. Certaines parce que l'auteur avait peur. Certaines parce qu'elles l'en ont empêché.

03   Fins
TERMINE LA PHRASE

Le jeu se termine par un terminal. Tu écris une phrase libre. Le jeu analyse ton texte et choisit une fin parmi cinq — selon le ton, les mots, ou ton silence.

RÉÉCRITURE
Le monde se stabilise. La Silhouette disparaît — terminée, pas effacée.
Mots positifs
EFFACEMENT
L'Archive se désintègre. Tout s'efface progressivement.
Mots négatifs
STAGNATION
Rien ne change. Le curseur recommence à clignoter.
Mots ambigus
CORRUPTION

Texte sans sens
ABANDON

04   Développement
COMMENT C'EST FAIT
Étape 01
IDÉE DE DÉPART
Le projet a commencé comme un simple platformer avec un chrono — une course entre deux joueurs. Puis l'idée a évolué vers un metroidvania narratif où les capacités débloquent de nouvelles zones, inspiré de Hollow Knight.
ConceptMetroidvaniaPlatformer
Étape 02
L'IDENTITÉ RED SCRIPT
L'univers a tout changé. Un monde où la réalité est faite de définitions écrites. Un personnage sans nom — un brouillon d'une phrase inachevée. Des fragments narratifs à collecter. Et une fin ouverte où le joueur écrit lui-même la conclusion du monde.
NarrationWorld BuildingGame Design
Étape 03
GODOT 4.5
Tout le jeu est développé avec Godot 4.5 sur Windows. Le moteur de mouvement inclut coyote time, jump buffer, double saut, dash avec traînée de fantômes, wall jump et grappin avec physique de pendule réelle. GDScript avec typage strict partout.
Godot 4.5GDScriptWindowsPixel Art 32x32
Étape 04
LE SYSTÈME DE FRAGMENTS
18 phrases inachevées dans 5 zones. Chaque fragment est un Area2D avec texte, animation de flottaison, PointLight2D et effet de collecte. Le HUD gèle le jeu et affiche le texte avec fondu. L'obscurité est gérée par CanvasModulate — les lumières percent le noir.
Area2DPointLight2DCanvasModulate
Étape 05
LE SCRIPT TERMINAL
La fin du jeu — une scène minimaliste avec un LineEdit où le joueur tape librement. L'algorithme analyse le ton et les mots clés pour choisir parmi 5 fins. L'effet machine à écrire lettre par lettre crée une atmosphère de terminal authentique.
LineEditAnalyse mots clés5 Fins
Étape 06
L'EFFET DE GLITCH
Toutes les 8 secondes, la Silhouette Rouge subit un glitch — elle tremble, devient rouge, puis 2 secondes après un message s'affiche : "Tu viens de devenir rouge. Ce n'est pas un bug. C'est ce que tu es." Un rappel constant que le personnage est incomplet.
